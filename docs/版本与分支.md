# 版本、分支与 Git 标签

## 分支策略

| 分支 | 含义 |
|------|------|
| **`main`** | **MVP v1.2 异步版（当前主线）**：`prepareProjectWorkflow` + Inngest + `executeProjectWorkflow`。2026-05-19 自 `feature/v1.2-inngest` 合并，议事见 [合并main-议事记录.md](./合并main-议事记录.md)。 |
| **`feature/v1.2-inngest`** | 历史开发分支；与 `main` 已 fast-forward 对齐，可保留作对照或删除远程分支（可选）。 |

**同步版回退**：标签 **`mvp-v1.1`**（合并前 `main`，提交 `76e5a3a` 一带），非 `main` 当前内容。

## 推荐 Git 标签

| 标签 | 建议指向 | 说明 |
|------|----------|------|
| **`mvp-v1.1`** | `main` 上含同步工作流的提交 | 最后一次「纯同步 generate」可运行快照。 |
| **`mvp-v1.2`** | 合并后 `main` 尖端 | 异步版生产主线（2026-05-19 起）。 |
| **`mvp-v1.2-code`** | 与 `mvp-v1.2` 同代或 feature 尖端 | 含 Inngest 改造后的代码快照（历史命名，可与 `mvp-v1.2` 并用）。 |

查看：`git show mvp-v1.1:lib/workflow.ts`（等）可对比两版差异。

## 风险备忘（与同步版的关系）

- `main` 为**替换式**异步架构；需要同步行为时请 **`git checkout mvp-v1.1`**（勿假设当前 `main` 仍为同步版）。
- 本机若 **3000 被占用**，工厂常用 **3001**：Next `npm run start -- -p 3001`（或 dev），Inngest：`npm run inngest:dev:3001`（见 `package.json`）。

## Inngest 事件

- 事件名：`project/generate.requested`
- `data` 仅含 `projectId`、`forceRegenerate`；业务数据一律从 Supabase 读取。

## `executeProjectWorkflow` 与过期事件

- 若事件到达时项目已是 **`completed`**，或状态为 **`pending` / `failed` 等非 `running`**：函数 **返回 `status: "skipped"`**，**不修改** `projects`（避免误把 `pending` 打成 `failed`）。
- 仅 **`running`** 时执行 8 个 Agent；执行中 LLM/DB 失败仍会走原有逻辑将项目标为 **`failed`**。
