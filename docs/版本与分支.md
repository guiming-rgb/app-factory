# 版本、分支与 Git 标签

## 分支策略

| 分支 | 含义 |
|------|------|
| **`main`** | **MVP v1.1 同步版**：`runProjectWorkflow` 长请求直连 8 Agent；适合无 Inngest 时的对照与回退。 |
| **`feature/v1.2-inngest`** | **MVP v1.2 异步版**：`prepareProjectWorkflow` + Inngest + `executeProjectWorkflow`；**不保留**同步 `runProjectWorkflow`。 |

合并 `feature/v1.2-inngest` 进 `main` 前，请先完成双进程真机验收。

## 推荐 Git 标签

| 标签 | 建议指向 | 说明 |
|------|----------|------|
| **`mvp-v1.1`** | `main` 上含同步工作流的提交 | 最后一次「纯同步 generate」可运行快照。 |
| **`mvp-v1.2-code`** | `feature/v1.2-inngest` 尖端提交 | 含 Inngest 改造后的代码快照。 |

查看：`git show mvp-v1.1:lib/workflow.ts`（等）可对比两版差异。

## 风险备忘（与同步版的关系）

- v1.2 分支是**替换式**异步架构，不是与同步版并存；需要同步行为时请 **checkout `main` 或使用标签 `mvp-v1.1`**。
- 本地 Inngest：`npm run inngest:dev` 默认 `http://localhost:3000/api/inngest`。若 Next 占用 **3001**，请改用：  
  `npx inngest-cli@latest dev -u http://localhost:3001/api/inngest`  
  或固定 Next：`npm run dev -- -p 3000`。

## Inngest 事件

- 事件名：`project/generate.requested`
- `data` 仅含 `projectId`、`forceRegenerate`；业务数据一律从 Supabase 读取。
