# MVP v5 — 记忆与技能 设计草案

> **状态**：v5-0 立项（2026-05-27）  
> **上位**：[执行计划.md](./执行计划.md) v5 行 · [项目功能与架构交接.md](./项目功能与架构交接.md)  
> **前置**：v4 Auth + RLS ✅

---

## 1. 目标与非目标

### 1.1 目标

| # | 目标 |
|---|------|
| G1 | **项目记忆**：用户可为项目添加持久化备注（偏好、约束、迭代反馈） |
| G2 | **工作流消费**：8 Agent 流水线可读最近记忆（v5-2） |
| G3 | **技能注册表**：`skills` 表有版本与发布状态，API 可读已发布技能 |
| G4 | **Agent 绑定**：`agents.skill_ids` 可解析为 prompt 片段（v5-4） |

### 1.2 非目标（v5 不做）

- 向量检索 / RAG 全库
- 技能市场 UI、付费、审核工作流
- 跨项目全局用户画像（v5.1+）

---

## 2. 现状

| 资产 | 状态 |
|------|------|
| `memories` 表 | schema 已有，RLS 仅 SELECT（v4-4） |
| `skills` 表 | schema 已有，无 seed 业务数据 |
| `agents.skill_ids` | jsonb 默认 `[]`，未接业务 |
| 工作流 | 不读 memories |

---

## 3. 数据模型（v5-1 增量）

### memories（沿用表结构）

| 列 | 用途 |
|----|------|
| `user_id` | 创建者 = `auth.uid()` |
| `project_id` | 归属项目 |
| `memory_type` | `note` \| `constraint` \| `feedback`（v5-1 字符串枚举） |
| `content` | 正文，≤2000 字 |
| `importance` | 1～5，默认 1 |

### skills（v5-3）

- 沿用 `status`: `draft` | `published`
- seed 2～3 条示例 published 技能

---

## 4. API（v5-1）

| 方法 | 路径 | 行为 |
|------|------|------|
| GET | `/api/projects/[id]/memories` | 列表，RLS |
| POST | `/api/projects/[id]/memories` | `{ content, memory_type?, importance? }` |
| DELETE | `/api/projects/[id]/memories/[memoryId]` | 仅创建者 |
