# R1 发行路线图

> 将 App 生产工厂生成的 App 从"可运行"推进到"可上架"的全平台发行指南。
> 最后更新：2026-06-16

---

## 概览

| 平台 | 当前状态 | R1 目标 | 依赖 |
|------|:--------:|---------|------|
| **macOS 桌面** | GHA 构建 .app，需"右键→打开" | 代码签名 + 公证后双击即用 | Apple Developer 账号 ($99/年) |

## 一、macOS 发行

### 1.1 准备工作

1. **加入 Apple Developer Program** ($99/年)
   - 访问 [developer.apple.com/programs](https://developer.apple.com/programs)
   - 用 Apple ID 注册，需提供身份信息

2. **生成 Developer ID Application 证书**
   - Xcode → Settings → Accounts → 登录 Apple ID
   - 或 [developer.apple.com/account](https://developer.apple.com/account) → Certificates
   - 创建 **Developer ID Application** 类型证书
   - 下载并双击导入钥匙串

3. **生成 App-Specific Password**
   - [appleid.apple.com](https://appleid.apple.com) → App-Specific Passwords
   - 生成一个密码（用于 notarytool 提交）

4. **导出证书为 p12**
   ```bash
   # 钥匙串中找到 Developer ID Application 证书
   # 右键导出为 .p12（需要设置导出密码）
   # Base64 编码
   base64 -i certificate.p12 | pbcopy
   ```

### 1.2 配置 GitHub Secrets

在 GitHub 仓库设置 → Secrets and variables → Actions 添加：

| Secret | 说明 |
|--------|------|
| `APPLE_DEVELOPER_ID` | 开发者 ID 名称，如 `"Developer ID Application: Your Name (TEAM123)"` 或直接用 Team ID |
| `APPLE_TEAM_ID` | Apple Developer Team ID（在开发者账号页面可找到，格式 `ABCD1234`） |
| `APPLE_ID_PASSWORD` | 上面生成的 App-Specific Password |
| `MACOS_CERT_BASE64` | 导出证书的 base64 编码 |
| `MACOS_CERT_PASSWORD` | 导出 p12 时设置的密码 |

GHA workflow 检测到这些 secrets 存在时自动执行签名 + 公证 + stapler。
secrets 不存在时跳过签名，行为与当前一致。

### 1.3 验证签名

```bash
# 本地验证（需要有 Mac）
codesign -dvvv /path/to/YourApp.app
spctl -a -v /path/to/YourApp.app
# 输出应为: accepted source=Notarized Developer ID
```

## 二、Windows 发行

### 2.1 准备工作

1. **购买代码签名证书**
   - EV Code Signing Certificate（推荐，$200-400/年）
   - 提供商：DigiCert、Sectigo、GlobalSign 等
   - EV 证书可立即获得微软 SmartScreen 信任

2. **导出证书为 pfx**
   ```bash
   # 从 Windows 证书管理器导出带私钥的 .pfx
   # Base64 编码
   certutil -encode certificate.pfx encoded.txt
   ```

### 2.2 配置 GitHub Secrets

| Secret | 说明 |
|--------|------|
| `WINDOWS_CERT_BASE64` | pfx 证书的 base64 编码 |
| `WINDOWS_CERT_PASSWORD` | pfx 证书密码 |

secrets 不存在时跳过签名。

### 2.3 验证签名

```bash
# Windows
signtool verify /v /pa /ph YourApp.exe
```

## 三、微信小程序发行

### 3.1 准备工作

1. **注册微信公众平台**
   - [mp.weixin.qq.com](https://mp.weixin.qq.com)
   - 选择"小程序"类型注册
   - 需要：**企业/个体工商户主体认证**（300 元认证费）

2. **获取正式 AppID**
   - 注册完成后在"开发"→"开发设置"中查看 AppID

3. **配置服务器域名**
   - 小程序后台 → 开发 → 开发设置 → 服务器域名
   - 添加 HTTPS 域名（需备案）：`https://your-supabase-project.supabase.co`
   - 微信小程序要求 **所有网络请求必须 HTTPS**

4. **准备隐私政策**
   - 使用工厂生成的隐私政策文档（见 `POST /api/generated-privacy`），或自备
   - 在微信小程序后台 → 设置 → 基本设置 → 用户隐私保护指引 中填写
   - 或在小程序代码中通过 `wx.openPrivacyContract` 接口展示

### 3.2 修改代码

在项目详情页或 codegen 调用时，设置微信 AppID：

```bash
# 方式 1：通过工厂界面（需 UI 支持）
# 方式 2：直接修改生成的 project.config.json
# 将 "appid": "touristappid" 替换为你的正式 AppID
```

### 3.3 Console 错误检查

提交前务必检查微信开发者工具的 Console 面板：

```
当前状况：6 errors / 7 warnings（2026-06-05 实测）
```

常见问题及解决方法：
- **`__dev__/xxx.js 错误`**：通常是第三方组件兼容性问题
- **`wx.xxx is not a function`**：基础库版本过低，检查 `libVersion`
- **`urlCheck` 相关**：测试时设为 `false`，正式版必须 `true` + 配置 HTTPS 域名
- **隐私政策弹窗不显示**：确保在 `app.json` 中配置 `"__usePrivacy__": true`（微信 2.32.3+）

### 3.4 提交流程

1. 微信开发者工具 → 上传 → 填写版本号和描述
2. 小程序后台 → 版本管理 → 提交审核
3. 审核周期约 1-7 个工作日
4. 审核通过后 → 发布

### 3.5 提审 Checklist

```
☐ 有正式 AppID（非 touristappid）
☐ 已配置服务器 HTTPS 域名
☐ 隐私政策已填写/展示
☐ Console 无红色错误（至少确认每个错误可忽略）
☐ 小程序类目已选择
☐ 测试号下所有功能正常
☐ 已上传代码到微信后台
```

## 四、鸿蒙发行

### 4.1 准备工作

1. **注册华为开发者**
   - [developer.huawei.com](https://developer.huawei.com)
   - 完成实名认证

2. **配置 AppGallery Connect**
   - [AppGallery Connect](https://developer.huawei.com/consumer/cn/service/jkos-platform)
   - 创建应用 → 获取 App ID 和证书

3. **生成签名文件**
   通过 DevEco Studio 生成：
   - DevEco Studio → Build → Generate Key and CSR
   - 生成 .p12（密钥库）和 .cer（证书请求文件）
   - 在 AGC 上传证书 → 下载 .p7b（Profile 文件）
   - 将三个文件放入工程 `signing/` 目录

### 4.2 配置 build-profile.json5

生成的工程中 `build-profile.json5` 已有签名配置占位：

```json5
"material": {
  "certPath": "./signing/developer.p7b",
  "keyStorePath": "./signing/developer.p12",
  "keyStorePassword": "your-keystore-password",
  "keyAlias": "key0",
  "keyPassword": "your-key-password",
  "profilePath": "./signing/developer.p7b",
  "signAlg": "SHA256withECDSA",
  "storePassword": "your-store-password"
}
```

填入实际的证书路径和密码。

### 4.3 构建 HAP/APP 包

```bash
# DevEco Studio 中
Build → Build HAP(s) / APP(s)

# 或在工程根目录
hvigorw assembleHap
```

### 4.4 上架 AGC

1. AGC → 我的应用 → 发布 → 软件包分发
2. 上传 HAP/APP 包
3. 填写应用描述、截图、分类
4. 提交审核

### 4.5 鸿蒙发行 Checklist

```
☐ 已注册华为开发者账号并实名认证
☐ 已在 AGC 创建应用
☐ 签名证书（.p12 + .cer + .p7b）已生成
☐ build-profile.json5 签名配置已填写
☐ 能成功构建 HAP 包
☐ 隐私政策已准备
☐ 已在 AGC 提交上架
```

## 八、发行路线图（建议顺序）

```
优先级 1  macOS 签名
  → 立即价值最大：消除 Gatekeeper，用户体验质变
  → 仅需 Apple Developer 账号 + 证书

优先级 2  微信小程序提审
  → 触达用户最广的平台
  → 需要主体认证（企业/个体户）

优先级 3  鸿蒙 AGC 上架
  → 华为生态用户
  → 需要 DevEco 构建体验

优先级 4  Windows 签名
  → 当前无 Gatekeeper 阻隔，不急
  → 等需要企业分发或商店上架时再配
```
