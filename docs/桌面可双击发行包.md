# Flutter 桌面可双击发行包（.app / .exe）

> **目标**：用户**无需安装 Flutter**，解压即可在 Mac 或 Windows 11 上双击运行工厂生成的 App。

---

## 一、三种获取方式

| 方式 | Mac .app | Win .exe | 谁装 Flutter |
|------|----------|----------|--------------|
| **A. 工厂本地构建机** | ✅ 在 Mac 上点「生成 Flutter」 | ✅ 在 Win11 上点「生成 Flutter」 | 仅**构建机**（可为你方 CI） |
| **B. GitHub Actions** | ✅ `macos-desktop.zip` | ✅ `windows-desktop.zip` | 仅 GitHub 云端 |
| **C. 生产 Vercel 网页** | ⏭ 默认无（仅源码 ZIP） | ⏭ 同上 | 用户**不用**装 |

生产站 https://app-factory-five.vercel.app **不能**在 Linux 服务器上编译 macOS/Windows 原生程序，因此网页生成默认只带：

- 完整 Flutter **源码 ZIP**
- 目录 **`双击运行-Desktop/README-双击运行.txt`**（说明如何拿到 .app/.exe）

若构建机成功，项目页会出现独立下载：**Mac .app**、**Win .exe 包**。

---

## 二、环境变量（工厂服务端）

| 变量 | 默认 | 说明 |
|------|------|------|
| `CODEGEN_DESKTOP_BUILD` | `auto` | `auto`：本机有 Flutter 且非 Vercel 时构建；`1` 强制；`0` 关闭 |
| `CODEGEN_DESKTOP_BUILD_TIMEOUT_MS` | `600000` | 单次 `flutter build` 超时 |

---

## 三、用户怎么用下载包

### macOS（笔记本 / iMac）

1. 下载 `*-macos.app.zip`
2. 解压得到 `xxx.app`
3. 双击打开（首次可能：系统设置 → 隐私与安全性 → 仍要打开）

### Windows 11

1. 下载 `*-windows.zip`
2. 解压整个 **Release** 文件夹
3. 双击其中的 `.exe`（需与 dll、data 等同目录）

---

## 四、GitHub Actions（推荐用于生产双平台）

1. 仓库 → **Actions** → **Flutter 桌面双平台构建** → **Run workflow**
2. 构建完成后下载 Artifacts：`macos-desktop`、`windows-desktop`
3. 分发给最终用户

工作流文件：`.github/workflows/flutter-desktop-dual-build.yml`

---

## 五、与鸿蒙 / 小程序

- **鸿蒙 ZIP**：不能在 Mac/Win 上当 .app/.exe 用。
- **小程序**：在微信内运行，不是独立 .exe/.app。

仅 **Flutter 栈** 提供 Mac/Windows 可双击发行包。
