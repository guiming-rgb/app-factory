# Flutter 桌面可双击发行包（.app / .exe）

> **目标**：用户**无需安装 Flutter**，解压即可在 Mac 或 Windows 11 上双击运行工厂生成的 App。

---

## 一、三种获取方式

| 方式 | Mac .app | Win .exe | 谁装 Flutter |
|------|----------|----------|--------------|
| **A. 工厂本地构建机** | ✅ 在 Mac 上点「生成 Flutter」 | ✅ 在 Win11 上点「生成 Flutter」 | 仅**构建机** |
| **B. GitHub Actions（推荐生产）** | ✅ 自动 | ✅ 自动 | 仅 GitHub 云端 |
| **C. 仅源码 ZIP** | ⏭ 无桌面包 | ⏭ 无 | 用户不用装 |

---

## 二、生产网页自动链路（Vercel）

1. 用户在项目页点击 **生成 Flutter**（同步，约 10–30 秒）。
2. 服务端写出 **源码 ZIP**，并（若已配置 token）`workflow_dispatch` 触发  
   `.github/workflows/flutter-desktop-dual-build.yml`。
3. **Inngest** 函数 `codegen-flutter-desktop-gha` 每 30s 轮询 GHA（最长约 20 分钟）。
4. 成功后把 `macos-{run_id}` / `windows-{run_id}` 产物写入 Storage，页面出现：
   - **Mac .app** → `/api/.../download?kind=macos`
   - **Win .exe 包** → `/api/.../download?kind=windows`

页面在 `metadata.desktopGha.status` 为 `queued` / `running` 时每 15s 自动刷新。

---

## 三、环境变量（Vercel / 本地）

| 变量 | 说明 |
|------|------|
| `GITHUB_DESKTOP_GHA_TOKEN` | PAT，需 **repo + actions + workflow** |
| `GITHUB_DESKTOP_GHA_REPO` | 默认 `guiming-rgb/app-factory` |
| `GITHUB_DESKTOP_GHA_WORKFLOW` | 默认 `flutter-desktop-dual-build.yml` |
| `GITHUB_DESKTOP_GHA_REF` | 默认 `main` |
| `CODEGEN_DESKTOP_GHA_ENABLED` | 设为 `1` 可在非 Vercel 环境强制启用 |
| `CODEGEN_DESKTOP_GHA_DISABLED` | 设为 `1` 关闭自动 GHA |
| `CODEGEN_DESKTOP_BUILD` | 本机构建：`auto` / `1` / `0`（Vercel 有 GHA 时跳过本机 build） |

检查：`npm run check:desktop:gha`

---

## 四、用户怎么用下载包

### macOS

1. 下载 `*-macos.app.zip`
2. 解压得到 `xxx.app`
3. 双击（首次可能需在「隐私与安全性」中允许）

### Windows 11

1. 下载 `*-windows.zip`
2. 解压整个 **Release** 目录
3. 双击其中的 `.exe`（需与 dll、data 等同目录）

---

## 五、手动触发 GHA（调试）

仓库 → **Actions** → **Flutter 桌面双平台构建** → **Run workflow**  
需填写 `spec_b64`、`run_id`、`app_name`（与网页触发相同）。

---

## 六、与鸿蒙 / 小程序

- **鸿蒙 ZIP**：不能在 Mac/Win 上当 .app/.exe 用。
- **小程序**：在微信内运行。

仅 **Flutter 栈** 提供 Mac/Windows 可双击发行包。
