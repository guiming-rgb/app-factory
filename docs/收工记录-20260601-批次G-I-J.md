## 必须开发者真机测试项

以下功能仅有自动化测试还不够，需要开发者在实际设备上验证：

1. **Flutter 桌面包 (Mac .app / Win .exe)**
   - 在 macOS 和 Windows 11 上双击运行生成的 .app 和 .exe
   - 验证 Supabase 连接是否正常、UI 是否完整
   - `npm run codegen:flutter` 生成后执行 `cd <project>/macos && flutter build macos`

2. **微信小程序真机预览**
   - 在微信开发者工具中打开生成的小程序项目
   - 扫码在手机上预览，验证 Tab 切换、列表加载、表单提交
   - `npm run codegen:wechat` 生成后用微信开发者工具打开

3. **鸿蒙 DevEco Studio 运行**
   - 在 DevEco Studio 中打开生成的鸿蒙项目
   - 连接到鸿蒙手机或模拟器，验证 Index 页和路由跳转
   - `npm run codegen:harmony` 生成后用 DevEco Studio 打开

4. **GitHub Actions 桌面包构建**
   - Push 代码到 GitHub → 触发 `.github/workflows/flutter-desktop-dual-build.yml`
   - 等待构建完成后下载 Mac/Win ZIP 验证
   - 需要仓库配置 `DESKTOP_GHA_PAT` 和 `DESKTOP_GHA_REPO`

5. **OBD-II / BLE 医疗设备连接**（若有硬件）
   - 生成的 BLE 扫描页面连接到真实 ELM327/血压计/血糖仪
   - 验证 GATT service UUID 过滤和数据解析

6. **生产环境 Inngest 队列**
   - Vercel 部署后检查 Inngest Cloud 是否正常消费事件
   - 创建项目 → 启动 Agent → 等待完成 → 检查 Sentry 无错误
   - `npm run verify:prod https://your-app.vercel.app`
