# 批量执行计划 — v4 生产验收 + v5 启动（2026-05-27）

> **原则**：[agent-testing-minimal-human.mdc](../.cursor/rules/agent-testing-minimal-human.mdc) — 能自动测的先测；仅维护者密钥/首次 SQL/浏览器 GUI 才请人。

---

## 一、本轮 Agent 自动执行（一条命令）

```bash
npm run verify:v4:batch          # 静态门禁 + 生产 HTTP 探针
npm run verify:v4:batch:local    # 上述 + 本地 Supabase check:v4:*（有 .env.local 时）
npm run build
```

| 脚本 | 测什么 | 需维护者？ |
|------|--------|------------|
| `verify:v4:owner-api` | 13 个 [id] 路由 owner 守卫 | 否 |
| `verify:v4:inngest-auth` | Inngest userId 接线 | 否 |
| `verify:v4:rate-limit` | generate/codegen 限流接线 | 否 |
| `verify:v4:production` | 生产 deploy/login/未登录 401 探针 | 否 |
| `verify:v4:production:rls` | 双用户 RLS（Supabase 直连） | **可选** env 测账号 |
| `check:v4:owner` / `rls` / `rate-limit` | 库表/策略就绪 | 否（有 DATABASE_URL 时） |
| `verify:v3:production` | 全链路 generate+codegen（贵、慢） | 否，但 **不纳入默认 batch** |

---

## 二、维护者一次性（Agent 不能代做）

| # | 项 | 命令/位置 |
|---|-----|-----------|
| M1 | Vercel 配 `NEXT_PUBLIC_SUPABASE_ANON_KEY` + Redeploy | Dashboard / `deploy:vercel:env` |
| M2 | Supabase 迁移三连 | `db:apply:v4-owner` → `db:apply:v4-rls` → `db:apply:v4-rate-limit` |
| M3 | v5 记忆写策略 | `db:apply:v5-memories`（v5-1 代码合并后） |
| M4 | **可选** 生产 RLS 深测账号 | Vercel/本地 env：`V4_TEST_EMAIL` + `V4_TEST_PASSWORD`（及 B 账号） |

**无 M1～M2 时**：生产仍为「v3 行为」（无登录门控）；`verify:v4:production` 会 **WARN 而非 FAIL**。

## 三、v5 分阶段（与 v4 并行推进）

| 阶段 | 交付 | 验收 |
|------|------|------|
| **v5-0** | 设计草案 + 批量验收脚本 | 本文 + `verify:v4:batch` |
| **v5-1** | 项目记忆 CRUD API | `GET/POST /api/projects/[id]/memories` + owner/RLS |
| **v5-2** | 工作流注入记忆摘要 | generate 前 CEO Agent 见最近 N 条记忆 |
| **v5-3** | Skills 注册表 + 发布状态 | `GET /api/skills` · seed · status=draft/published |
| **v5-4** | Agent ↔ Skill 绑定 | `agents.skill_ids` 解析进 prompt |
| **v5-5** | 详情页记忆 UI | 项目页增删记忆（可选浏览器一次） |

**本轮代码目标**：v5-0 + v5-1 + v4 生产探针。
