## 〇、重启后 3 分钟恢复（维护者）

```bash
# 1. 进入工程（Secrets 在 .env.local，勿提交 Git）
cd "/Users/guiming/Desktop/app生产工厂/app-factory"

# 2. 若需 git push / 生产探针：先开本地代理（维护者常用端口 7897）
nc -zv 127.0.0.1 7897
export http_proxy=http://127.0.0.1:7897
export https_proxy=http://127.0.0.1:7897

# 3. 本地工厂（二选一）
npm run dev:codegen:3001          # 推荐：一键 Next + Inngest
# 或双终端：
#   A: npm run start -- -p 3001
#   B: npm run inngest:dev:3001     → Inngest UI http://127.0.0.1:8288

# 4. 浏览器
open http://localhost:3001
# 生产站（无需本地）：https://app-factory-five.vercel.app
```

| 重启后需手动开的服务 | 何时需要 |
|----------------------|----------|
| **HTTP 代理 7897** | `git push`、生产 curl 探针（`V3_HTTP_PROXY`） |
| **Next + Inngest 双进程** | 本地 8-Agent / 异步 codegen |
| **Docker Desktop** | 可选：Flutter Docker analyze（`CODEGEN_DOCKER_ANALYZE_*`） |
| **微信开发者工具** | 小程序模拟器 / 真机预览 |
| **DevEco Studio** | 鸿蒙模拟器 / 真机 Run |
| **Flutter / Xcode / VS** | 可选：改 Flutter 源码后 `flutter run` |

**不必重启后立刻开**：Vercel 生产站、GitHub Actions、Supabase（云端常开）。

### 1.3 `.env.local` 关键变量（名称 only，值在本地文件）

| 变量 | 必填 | 用途 |
|------|------|------|
| `NEXT_PUBLIC_SUPABASE_URL` | ✅ | Supabase 项目（工厂只用 **`dllaezdyxmoebkkwbftd`**） |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | ✅ | 前端 |
| `SUPABASE_SERVICE_ROLE_KEY` | ✅ | 服务端 / Storage |
| `OPENAI_API_KEY` | ✅ | Agent / Report→Spec |
| `OPENAI_BASE_URL` / `OPENAI_MODEL` | 建议 | 如 DeepSeek |
| `NEXT_PUBLIC_APP_URL` | 建议 | 本地 **`http://localhost:3001`** |
| **`INNGEST_DEV=1`** | 本地联调 ✅ | 无则 `PUT /api/inngest` **500** |
| `INNGEST_EVENT_KEY` / `INNGEST_SIGNING_KEY` | 生产 Cloud | 本地可空；**生产未配 → 桌面包靠刷新记录** |
| `GITHUB_PAT` | GHA / push | 需 **repo + actions**；push workflow 还需 **workflow** |
| `GITHUB_DESKTOP_GHA_REPO` | 可选 | 默认 `guiming-rgb/app-factory` |
| **`V3_HTTP_PROXY`** | 探针 | 如 `http://127.0.0.1:7897`（**勿提交**） |
| `V4_TEST_EMAIL` / `V4_TEST_PASSWORD` | 脚本 | E2E / 生产 verify（见下表测试账号） |

同步 Vercel：`npm run deploy:vercel:env` → `npm run deploy:vercel`

### 1.4 本地 vs 生产差异

| 能力 | 本地 Mac | Vercel 生产 |
|------|----------|-------------|
| 8-Agent + Inngest | ✅ 双进程 | ⚠️ 需 Inngest Cloud 密钥才稳 |

| Mac/Win **桌面包** | 本机可编（可选） | **GHA** 云端编 |

| Win 包下载 | — | 工厂 **Win .exe 包** 直链（~12MB） |

### 4.1 产品战略（已定）

| 决策 | 内容 |

### 4.2 架构构思（当前可用链路）

```
工厂 Web 生成 Flutter
  → 源码 ZIP（Supabase Storage）
  → workflow_dispatch GHA（Vercel 上 GITHUB_PAT）
  → macos-latest / windows-latest 构建
  → 刷新 Codegen 记录 sync-desktop-gha
  → 页面：Win .exe 包 + Mac .app(GitHub)
  → 用户：Win 直下；Mac 去 Artifacts ~51MB

工厂 Web 生成小程序
  → 下载 ZIP → 微信开发者工具（测试号）→ 模拟器/预览/提审

工厂 Web 生成鸿蒙
  → 下载 ZIP → DevEco → 鸿蒙设备 Run
```
