## 二、Q2 2026（7-9月）— 深度体验

### 目标：生成的 App 从「能跑」提升到「能用」— 用户拿到 ZIP 后不需改代码就能直接使用

### 🔴 Q2-M1（7月）：行业 Widget 补齐 + 模板渲染引擎

**为什么先做这个**：15/20 行业生成的 Flutter App 使用通用 list/detail 页，没有行业专属 UI 组件。这是「生成 App 长得像模板」的根本原因。

**关键决策**：放弃 TypeScript emit 字符串方案 → 改用 **Dart 模板文件 + Mustache 变量替换**。

```
方案对比：
❌ emit 字符串 (当前方案):
   - 无法用 dart analyze 检查模板本身
   - 重构困难（TypeScript 里写 Dart 字符串）
   - IDE 无语法高亮
   - linter 会回退过长的模板字符串

✅ Dart .mustache 模板文件:
   - 模板本身可被 dart analyze 验证
   - IDE 有 Dart 语法高亮和自动补全
   - 可独立测试（拿模板 + mock 数据渲染）
   - 不会被 linter 回退
```

**执行清单**：

| # | 任务 | 产出文件 | 验收标准 |
|---|------|---------|---------|
| 1 | 选型 Mustache 渲染（推荐 Node `mustache`） | `lib/codegen/template-renderer.ts` | 传入模板路径+变量 → 输出渲染后字符串 |
| 2 | 将现有 5 个 emit 函数迁移到模板文件 | `templates/flutter-minimal/lib/core/widgets/industry/*.dart.mustache` (5 个) | `dart analyze` 零 error |
| 3 | 补齐 15 个缺失行业的 Widget 模板（对标一线 App） | 同上目录 (15 个新文件) | 每个文件 ≥ 2 个独立组件，≥ 80 行 Dart |
| 4 | `copyIndustryTemplate` 改为调用 Mustache 渲染 | `lib/flutter-codegen/copy-industry-template.ts` | 19 行业生成后 `dart analyze` 全通过 |
| 5 | 新建门禁 `verify:industry:widgets` | `scripts/verify-industry-widgets.mjs` | 19 行业 × dart analyze = 全绿 |
