## 一、首版 `wechat-miniprogram-minimal`（✅ v2b-实现-1 已落盘）

| 分类 | 支持 | 不支持（首版） |
|------|------|----------------|
| **页面** | 2～5 个原生页（Tab + 列表/详情占位） | 复杂自定义组件库、小游戏 Canvas 重度 |
| **导航** | `tabBar`、页面栈跳转 | 全屏广告、复杂嵌套导航 |
| **数据** | `wx.request` → Supabase REST（只读为主） | 离线包、复杂同步 |
| **登录** | `wx.login` 占位 + 文档说明后端换 session | 完整 unionId 用户体系（需后端立项） |
| **支付** | 文案 limitations 声明不支持 | `wx.requestPayment` |
| **直播/音视频** | 拒绝或 limitations | 直播插件、RTC |
| **地图** | 静态图占位 | 重度 LBS、导航 |
| **分包** | `subpackages` 空壳配置 | 大型分包自动拆分 |
| **云开发** | 可选文档说明；首版倾向 **HTTPS + Supabase** | 强绑云开发数据库 |

## 二、与 Flutter 矩阵的差异

| 能力 | Flutter v2a | 小程序 v2b |
|------|-------------|------------|
| 商店发布 | App Store / 应用市场 | 微信公众平台审核 |
| 登录 | Supabase Auth SDK | `wx.login` 必须 |
| 分享 | 系统分享 SDK | `onShareAppMessage` 需单独生成 |
| 代码 | Dart | WXML + JS |

**禁止**：在 Flutter Spec 的 `targets.flutter` 下写小程序能力；须 `targets.wechatMiniProgram` 或 limitations。

## 三、Validator 拒绝对照（规划）

以下写入 Spec 且未标 `templateLimited` 时，Validator 应 **warn 或 fail**：

- `payment` / `wx.requestPayment`  
- `live` / `channel` 直播  
- `minigame` 重度游戏逻辑
