## 五、数据侧检疫（Supabase SQL Editor，必做）

将下面三处 **`'<你的 projects.id>'`** 换成「一、元信息」中的 UUID 后执行。

```sql
-- A. 项目态 + final_report 长度
select id, status, length(coalesce(final_report, '')) as final_report_len
from projects
where id = '<你的 projects.id>';

-- B. agent_runs 条数（成功路径应为 8）
select count(*) as agent_run_rows
from agent_runs
where project_id = '<你的 projects.id>';

-- C. 各 agent 状态（排错）
select agent_id, status, count(*) as n
from agent_runs
where project_id = '<你的 projects.id>'
group by agent_id, status
order by agent_id;
```

- [ ] **A**：`status = 'completed'` 且 `final_report_len` **> 0**（或你认可的阈值）
- [ ] **B**：`agent_run_rows = 8`
- [ ] **C**：成功路径下每条对应 **`status = 'completed'`**（与 [开发纲要.md](./开发纲要.md) 2.1 一致）
- [ ] 无异常重复 `agent_runs`（除主动 **`forceRegenerate`** 触发的全量重跑）

## 六、v1.1 行为抽测（必做）

- [ ] **历史列表**：`http://localhost:3000/projects` 可见该项目
- [ ] **导出**：下载 Markdown 正常（`GET /api/projects/[id]/export`）
- [ ] **复制报告**：按钮可用（localhost 或 HTTPS）
- [ ] **409**：项目在 **`running`** 时再次点击「启动生产」→ 应 **409** 或等价防重复行为
