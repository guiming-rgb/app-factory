## 二、运行环境（App 生产工厂）

### 1. 谁需要装什么

| 角色 | 环境 | 说明 |
|------|------|------|
| **维护者 / 开发工厂** | Mac 或 Windows | Node 20+ · Git · 浏览器 |
| **只用生产站** | 任意 OS + 浏览器 | 无需 Node/Flutter |
| **本地联调 8-Agent** | Mac/Win | 双进程：Next + Inngest Dev（`INNGEST_DEV=1`） |

### 2. 本地工厂 `.env.local`（复制自 `.env.local.example`）

| 变量 | 必填 | 用途 |
|------|------|------|
| `NEXT_PUBLIC_SUPABASE_URL` | ✅ | Supabase |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | ✅ | 前端 |
| `SUPABASE_SERVICE_ROLE_KEY` | ✅ | 服务端 / Storage |
| `OPENAI_API_KEY` | ✅ | 报告→Spec / Agent |
| `OPENAI_BASE_URL` / `OPENAI_MODEL` | 建议 | 如 DeepSeek |
| `NEXT_PUBLIC_APP_URL` | 建议 | 本地 `http://localhost:3001` |
| `INNGEST_DEV=1` | 本地联调 ✅ | 无则 `/api/inngest` 易 500 |
| `INNGEST_EVENT_KEY` / `INNGEST_SIGNING_KEY` | 生产 Cloud | 本地可空 |
| `GITHUB_PAT` | GHA/推仓库 | 需 **repo + actions**；push workflow 还需 **workflow** |
| `GITHUB_DESKTOP_GHA_REPO` | 可选 | 默认 `guiming-rgb/app-factory` |
| `V3_HTTP_PROXY` | 探针 | 如 `http://127.0.0.1:7897`（勿提交） |

**同步到 Vercel**：`npm run deploy:vercel:env` → `npm run deploy:vercel`

## 四、端到端链路（当前可用）

```
工厂 Web 生成 Flutter
  → 源码 ZIP（Storage）
  → 触发 GHA（GITHUB_PAT on Vercel）
  → Mac/Win 构建（GitHub Actions #3 类成功）
  → 刷新记录 sync-desktop-gha
  → 页面上：Win .exe 包 + Mac .app(GitHub)
  → 用户：Win 直下；Mac 去 Artifacts 下 ~51MB
```

```
工厂 Web 生成小程序
  → 下载 ZIP → 微信开发者工具导入（测试号）→ 编译 → 模拟器/预览
```
