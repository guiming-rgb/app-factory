# S6 — 发版前全链路验收

> **状态**：脚本 ✅（2026-05-22）· 全链路需在 **Inngest + 网络可达** 环境执行

## 命令

| 命令 | 场景 |
|------|------|
| `npm run verify:s6:release` | **推荐**：build → batch:local → 生产探针 → 生产全链路（失败则本地 fallback） |
| `npm run verify:v3:production` | 仅生产：创建项目 → 8-Agent generate → wechat codegen |
| `npm run verify:s6:local-full` | 仅本地：需 `3001` + `inngest:dev:3001` |

## 前置

### 生产（Vercel）

- Vercel 已 Redeploy，且 `deploy/status` 全部 `ok`（含 `inngest_cloud`、`anon_key`）
- `.env.local` 测试账号（与生产 Supabase 同一项目）：
  ```bash
  V4_TEST_EMAIL=e2e-local@app-factory.test
  V4_TEST_PASSWORD=...
  ```

### 本地 fallback

```bash
# 终端 A
npm run start -- -p 3001

# 终端 B
npm run inngest:dev:3001

# 终端 C
npm run verify:s6:local-full
```

## v4 Auth 说明

生产已启用 `NEXT_PUBLIC_SUPABASE_ANON_KEY` 时，全链路脚本会先 **Supabase 登录** 再带 Cookie 调 API；未配置 anon key 时退化为匿名（旧 v3 行为）。

## 网络

本机 curl/fetch 访问 Vercel 可能超时；`verify:s6:release` 会在生产 unreachable 时自动尝试 **本地全链路**。发版前仍建议在浏览器确认 `/login` 与一次手动 generate。
