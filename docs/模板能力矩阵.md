# 模板能力矩阵（v2 — 20260531 更新）

> **用途**：对照 App Spec 与用户描述，超出能力时打 `complianceFlags.templateLimited` 并生成人工清单。  
> **维护**：Codegen / Spec 变更时同步更新。

## Flutter 模板（`templates/flutter-minimal`）

| 支持 ✅ | 不支持 ❌（须降级或人工清单） |
|--------|---------------------------|
| Tab + 列表 + 详情 + 表单 | 复杂地图、蓝牙、AR |
| **待办清单 MVP**（添加/完成/删除，本地持久化） | 实时音视频、直播、WebRTC |
| Supabase Auth **登录/注册页面** | 应用内支付（需资质） |
| **实体 CRUD** + Supabase REST 查询 + 下拉刷新 | 大型离线同步 |
| **实体详情页**（字段展示 + Supabase fetch） | 复杂图表、可视化（需 chart 库） |
| **表单录入**（字段类型自动映射控件） | 推送通知 |

| **实体关系**（外键 DDL + 关联字段） | 机器学习 / AI 推理 |
| 响应式布局（手机/平板断点） | 第三方 SDK 集成 |

## 规则

1. 用户描述明显超出上表 → Spec 设 `templateLimited: true`，CEO/PRD 须列出「需人工开发」项。  
2. **禁止**硬编不可靠原生插件或虚假「全功能」承诺。  
3. 生产 Flutter codegen：`analyzeEnvironment=vercel-no-docker` 为**预期**，不视为缺陷。
4. Form 页面：非 Supabase 环境时优雅降级为本地 Toast 提示。
5. Auth 页面：生产环境的 Supabase URL/Anon Key 通过 `env.dart`/`supabase.js` 注入。
