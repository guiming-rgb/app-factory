# 模板能力矩阵

> **用途**：App Spec 与 Generator **不得**对外承诺超出本表的能力；超出须 `complianceFlags.templateLimited: true` + `limitations[]`。  
> **配套**：[多平台App生产工厂路线图.md](./多平台App生产工厂路线图.md) · [v2a-Flutter-最小模板-目录结构.md](./v2a-Flutter-最小模板-目录结构.md)

---

## 一、Flutter 最小模板（v2a 首版）

| 分类 | 支持 | 不支持（首版） |
|------|------|----------------|
| **平台** | iOS、Android（同一工程） | 桌面 Win/macOS/Linux 发布 |
| **形态** | 手机 + 平板断点（`layoutRules`） | 手表、TV |
| **导航** | 底部 Tab、列表→详情、简单表单 | 复杂 Drawer 多级、深链全量 |
| **数据** | Supabase 表 CRUD（anon + 未来 Auth） | 复杂离线同步、图数据库 |
| **认证** | Supabase Auth 占位（email / magic link 可扩展） | 微信登录（属 **v2b 小程序**） |
| **UI** | Material 3 基础组件 | 重度自定义动画、游戏引擎 |
| **商业** | 展示「需人工接入」占位 | 应用内支付、订阅 SDK |
| **媒体** | 静态图占位 | 短视频、直播、RTC |
| **设备** | — | 蓝牙、NFC、AR、后台定位 |

## 二、微信原生小程序（v2b · 必选，本表仅列边界）

| 分类 | 规划支持（v2b 立项后填实现列） | 备注 |
|------|-------------------------------|------|
| 登录 | `wx.login` + 后端换 session | 与 Flutter **独立工程** |
| 页面 | 原生 WXML 页，由 Spec `targets.wechatMiniProgram` 驱动 | 非 WebView 套壳 |
| 后端 | HTTPS 调 Supabase REST / Edge Functions | 须备案域名 |

**状态**：v2b **未开始**；工厂对外须写「小程序能力未就绪」而非「不需要」。

## 四、校验规则（Validator 自定义层）

当用户 Spec 含以下关键词或字段组合时，**必须**触发 `templateLimited` 或拒绝 codegen：

- `payment` / `iap` / `stripe` / `alipay`  
- `live_stream` / `rtc` / `agora`  
- `bluetooth` / `ibeacon`  
- `miniprogram` 能力写在 Flutter 目标下（应改 `targets.wechatMiniProgram`）
