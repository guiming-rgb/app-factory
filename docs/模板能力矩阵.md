# 模板能力矩阵（首版）

> **用途**：对照 App Spec 与用户描述，超出能力时打 `complianceFlags.templateLimited` 并生成人工清单。  
> **维护**：Codegen / Spec 变更时同步更新。

---

## Flutter 模板（`templates/flutter-minimal`）

| 支持 | 不支持（须降级或人工清单） |
|------|---------------------------|
| Tab + 列表 + 详情 + 表单 | 复杂地图、蓝牙、AR |
| **待办清单 MVP**（添加/完成/删除，本地） | 云同步、复杂 CRUD |
| Supabase Auth 登录/注册模式 | 应用内支付（需资质） |
| 基础 CRUD + Supabase 查询 | 抖音级短视频、直播、IM |
| 响应式布局（手机/平板断点） | 大型离线同步、复杂音视频 |
| 本地 analyze 门禁（Docker 可用时） | Vercel 生产无 Docker → analyze 跳过 |

---

## 微信小程序模板（`templates/wechat-miniprogram-minimal`）

| 支持 | 不支持 |
|------|--------|
| TabBar + 首页列表占位 + 隐私弹窗 | 云开发完整后端 |
| **待办清单 MVP**（index 页） | 云同步、复杂 CRUD |
| 由 Spec `resolveCodegenScreens` 生成多 pages | 分包、插件、直播组件 |
| wcc/wcsc 真编译门禁（本地） | 微信审核类目自动匹配 |
| 与 App Spec `wechatMiniProgram` 段对齐 | UGC 审核、支付 |

---

## 鸿蒙模板（`templates/harmony-minimal`）

| 支持 | 不支持 |
|------|--------|
| Spec→多 `.ets` 页面（`main_pages.json`） | DevEco/hvigor 远程编译 |
| Index + Tab 对应页占位 | 复杂 ArkUI 动画、分布式能力 |
| **待办清单 MVP**（添加/完成/删除，本地） | 云同步、登录、复杂 CRUD |
| 结构门禁（`harmony-structure`） | 应用市场签名/上架 |
| bundleName 由 `appName` snake_case 推导 | 支付、推送 SDK |

---

## 后端（Supabase 优先）

| 支持 | 后置 |
|------|------|
| Auth + PostgreSQL + Storage | NestJS 独立后端（BackendTarget 抽象已预留） |
| RLS 按 `owner_id` 隔离 | 多租户 B2B |

---

## 规则

1. 用户描述明显超出上表 → Spec 设 `templateLimited: true`，CEO/PRD 须列出「需人工开发」项。  
2. **禁止**硬编不可靠原生插件或虚假「全功能」承诺。  
3. 生产 Flutter codegen：`analyzeEnvironment=vercel-no-docker` 为**预期**，不视为缺陷。
