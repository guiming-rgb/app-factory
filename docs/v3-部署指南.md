# MVP v3 — 部署闭环指南

> **范围**：Next.js 工厂上云（Vercel）+ Inngest Cloud + Supabase；不含 Auth/RLS（见 v4）。

## 1. 前置条件

| 项 | 说明 |
|----|------|
| Supabase | 工厂项目 **`dllaezdyxmoebkkwbftd`**；已跑 `schema.sql`、`20260520_codegen_runs.sql` |
| GitHub | 代码在 `guiming-rgb/app-factory` |
| Inngest Cloud | 免费档即可；注册 [inngest.com](https://www.inngest.com) |
| Vercel | 绑定 GitHub 仓库 |

本地自检：

```bash
npm run check:deploy
npm run build
```

## 2. Vercel 部署（维护者网页操作）

1. 打开 [vercel.com/new](https://vercel.com/new) → Import `guiming-rgb/app-factory`
2. Framework Preset：**Next.js**（自动）
3. **Environment Variables**（Production + Preview 均填）：

| 变量 | 说明 |
|------|------|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase Project URL |
| `SUPABASE_SERVICE_ROLE_KEY` | Service role（勿加 NEXT_PUBLIC_） |
| `OPENAI_API_KEY` | 模型 Key |
| `OPENAI_BASE_URL` | 如 `https://api.deepseek.com` |
| `OPENAI_MODEL` | 如 `deepseek-chat` |
| `NEXT_PUBLIC_APP_URL` | 部署后的域名，如 `https://app-factory-xxx.vercel.app` |
| `INNGEST_EVENT_KEY` | Inngest Cloud → Manage → Keys |
| `INNGEST_SIGNING_KEY` | 同上 |
| `SUPABASE_STORAGE_BUCKET` | 默认 `codegen-artifacts`（可选） |

**不要**在生产环境设置 `INNGEST_DEV=1`。

4. Deploy → 等待 Build 成功
5. 复制 Production URL，回填 `NEXT_PUBLIC_APP_URL` 并 Redeploy 一次

## 3. Inngest Cloud 同步

1. Inngest Dashboard → **Apps** → Create App
2. **Serve URL**：`https://<你的-vercel-域名>/api/inngest`
3. 确认函数列表含 `codegen-flutter`、`codegen-wechat`、`project-generate` 等
4. 发送测试事件或从工厂 UI 触发一次 generate / codegen 验证消费

## 4. 代码生成与 Docker analyze 说明

| 环境 | Docker analyze |
|------|----------------|
| **本地双进程** | ✅ 默认在 codegen 完成后跑 `dart analyze`（需 Docker Desktop） |
| **Vercel Serverless** | ⏭ 跳过（无 Docker）；ZIP 仍生成，metadata 为 `analyzeStatus: skipped` |

若希望云端也跑 analyze，需另部署 **带 Docker 的 Worker**（Fly.io / 自建 VM），Inngest 指向该 Worker 的 `/api/inngest`——属 v3.1 扩展，本指南不展开。

环境变量：

```bash
# 本地关闭 analyze 门禁
CODEGEN_DOCKER_ANALYZE_DISABLED=1

# analyze 失败仍标记 completed（仅记录 metadata，默认 strict 会 failed）
CODEGEN_DOCKER_ANALYZE_STRICT=0
```

## 5. 部署后验收清单

- [ ] 首页可创建项目
- [ ] 「启动 AI 后台生产」→ Inngest 消费 → `completed`
- [ ] 「后台生成 Flutter ZIP」→ 详情页 **已完成** + 下载 ZIP
- [ ] `npm run verify:v13 -- <projectId>`（本地对同库项目）或 Dashboard 看 `usage_logs`
- [ ] Inngest Dashboard 无持续 failed 事件
