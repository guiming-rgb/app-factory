# 发布指南

将生成的 App 发布到各大应用商店需要满足特定的技术要求和审核标准。本文档提供详细的发布检查清单和指导。

---

## App Store Connect 提交

### 前提条件

- Apple Developer 账号（$99/年）
- 生成的 Flutter 项目
- macOS 和 Xcode 环境

### 准备工作

#### 1. 证书和标识符

在 [Apple Developer Portal](https://developer.apple.com) 配置：

```bash
# Bundle Identifier 需唯一，如 com.yourcompany.yourapp
# 建议格式：com.{company}.{appName}
```

#### 2. 在 App Store Connect 创建应用

1. 登录 [App Store Connect](https://appstoreconnect.apple.com)
2. 点击"我的 App" → "+" → "新建 App"
3. 填写：
   - 平台：iOS
   - 名称：与 App Spec 的 `displayName` 一致
   - 语言：简体中文（或目标语言）
   - Bundle ID：选择已注册的标识符
   - SKU：自定义唯一标识

#### 3. 填写应用信息

- **描述**（最多 4000 字符）：应用功能说明
- **关键词**（最多 100 字符）：ASO 关键词，逗号分隔
- **技术支持 URL**：必须可访问
- **营销 URL**：可选
- **隐私政策 URL**：**必须提供**（见下方）

### 截图要求

| 设备 | 尺寸 | 数量 | 格式 |
|------|:----:|:----:|:----:|
| 6.5 英寸（iPhone 14 Pro Max） | 1242 × 2688 | 3-5 张 | PNG/JPG |
| 5.5 英寸（iPhone 8 Plus） | 1242 × 2208 | 3-5 张 | PNG/JPG |
| iPad Pro (12.9 英寸) | 2048 × 2732 | 3-5 张 | PNG/JPG |

**生成截图的建议：**

- 使用模拟器运行 `flutter run` 后通过 Xcode 截图
- 确保截图中使用真实数据而非占位符
- 包含应用的主要功能页面

### 提交审核清单

- [ ] Bundle Identifier 与证书匹配
- [ ] 应用图标（1024×1024 PNG，无圆角）
- [ ] 截图已上传（至少 3 张）
- [ ] 应用描述已填写
- [ ] 隐私政策 URL 可访问
- [ ] 技术支持 URL 可访问
- [ ] 版权信息已填写
- [ ] 分级（如 4+ / 12+ / 17+）
- [ ] 价格与销售范围已设置
- [ ] 构建版本已上传（`flutter build ipa`）
- [ ] 出口合规信息已填写（通常选"否"）
- [ ] 内容版权声明已确认

### 常见拒绝原因

| 拒绝原因 | 解决方案 |
|---------|---------|
| 应用崩溃 | 在真机上充分测试后再提交 |
| 界面不完整 | 确保无"开发中"或"即将推出"页面 |
| 登录要求不合理 | 允许用户在不登录的情况下浏览核心内容 |
| 隐私政策不足 | 提供完整的隐私政策页面 |
| 使用私有 API | 检查第三方库是否使用了私有 API |
| 占位符内容 | 替换所有示例/占位符数据 |

## Google Play 提交

### 前提条件

- Google Play 开发者账号（$25 一次性费用）
- 生成的 Flutter 项目

### 准备工作

#### 1. 生成签名密钥

```bash
# 创建 keystore（首次）
keytool -genkey -v -keystore upload-keystore.jks \
  -alias upload -keyalg RSA -keysize 2048 \
  -validity 10000

# 配置 android/key.properties
storePassword=your_password
keyPassword=your_password
keyAlias=upload
storeFile=../upload-keystore.jks
```

#### 2. 在 Google Play Console 创建应用

1. 登录 [Google Play Console](https://play.google.com/console)
2. 点击"创建应用"
3. 填写应用名称、默认语言、应用或游戏、免费或付费

#### 3. 填写商店信息

- **应用名称**（最多 50 字符）
- **简短描述**（最多 80 字符）
- **完整描述**（最多 4000 字符）
- **分类**：选择合适的应用类别
- **标签**：最多 5 个
- **联系方式**：电子邮件、网站、电话

### 截图要求

| 类型 | 数量 | 规格 |
|------|:----:|------|
| 手机截图 | 2-8 张 | 16:9 或 9:16，3200×3200 以内 |
| 平板截图 | 2-8 张 | 同上 |
| 应用图标 | 1 张 | 512×512 PNG，带 32×32 圆角 |
| 特色图 | 1 张 | 1024×500 JPG/PNG |

### 提交审核清单

- [ ] 应用签名密钥已配置
- [ ] AAB 构建成功（`flutter build appbundle`）
- [ ] 应用描述已填写
- [ ] 截图已上传
- [ ] 隐私政策 URL 可访问
- [ ] 应用分类已选择
- [ ] 目标年龄段已设置
- [ ] 价格已设置
- [ ] 国家和地区已选择
- [ ] 广告声明已配置（如有广告）

## 微信小程序审核

### 前提条件

- 微信小程序 AppID（已在微信公众平台注册）
- 通过微信认证（300 元/年认证费，可选但推荐）

### 提交前检查

#### 1. 基础配置

在 [微信公众平台](https://mp.weixin.qq.com/) 小程序管理后台：

- [ ] 基本设置 → 小程序名称、头像、简介已填写
- [ ] 服务器域名已配置（request / uploadFile / downloadFile / socket）
- [ ] 业务域名已配置（web-view 使用）
- [ ] 类目已选择（最多 5 个）
- [ ] 隐私保护指引已设置

#### 2. 代码检查

- [ ] `app.json` 中所有页面路径正确
- [ ] 所有网络请求使用 HTTPS
- [ ] `request` 合法域名已在后台配置
- [ ] 用户隐私接口已声明用途
- [ ] 登录流程完整（wx.login → code → 后端登录）
- [ ] 无 wx.开头的未声明 API 调用

#### 3. 内容检查

- [ ] 无"测试"、"示例"等占位文本
- [ ] 图片资源完整
- [ ] 隐私政策页面可访问
- [ ] 所有功能可正常使用
- [ ] 无违法违规内容

### 提交审核

1. 在微信开发者工具中点击 **上传**
2. 填写版本号和版本描述
3. 登录微信公众平台 → 版本管理
4. 找到上传的版本 → 提交审核
5. 填写审核信息：
   - 功能描述
   - 测试账号（如需登录）
   - 测试说明

### 审核周期

| 提交时间 | 预计审核时间 |
|---------|------------|
| 工作日 9:00-18:00 | 1-3 小时 |
| 工作日 18:00 后 | 次日上午 |
| 周末/节假日 | 下一个工作日 |

### 常见拒绝原因

| 拒绝原因 | 解决方案 |
|---------|---------|
| 类目与内容不符 | 检查并修改类目选择 |
| 缺少必需类目资质 | 上传相关资质文件 |
| 功能不完整/无实质功能 | 补充缺失功能 |
| 界面设计简陋 | 优化 UI 设计 |
| 存在未登录可操作的支付/诱导行为 | 修复逻辑 |
| 隐私政策链接无效 | 确保隐私政策页面可访问 |

## 华为应用市场提交

### 前提条件

- 华为开发者账号（已实名认证）
- DevEco Studio 和鸿蒙 SDK

### 准备工作

#### 1. 签名配置

在 DevEco Studio 中配置签名：

1. **Build → Generate Key and CSR**
2. 填写证书信息
3. 生成 `.cer` 和 `.p12` 文件
4. 在 AppGallery Connect 中上传证书

#### 2. 创建应用

在 [AppGallery Connect](https://developer.huawei.com/consumer/cn/service/josp/agc/index.html)：

1. 点击"我的应用" → "新建应用"
2. 选择 HarmonyOS 平台
3. 填写应用名称和包名
4. 选择应用分类

### 资料要求

| 项目 | 要求 |
|------|------|
| 应用名称 | 1-30 字符 |
| 应用描述 | 20-2000 字符 |
| 应用图标 | 216×216 PNG，圆角 48×48 |
| 截图 | 至少 2 张，JPG/PNG，比例 16:9 或 9:16 |
| 隐私政策 | 必须提供 |
| 版权信息 | 如有需要 |

### 提交审核清单

- [ ] 签名文件已配置
- [ ] HAP 包构建成功
- [ ] 应用描述已填写
- [ ] 截图已上传（至少 2 张）
- [ ] 隐私政策 URL 可访问
- [ ] 应用分类已选择
- [ ] 目标用户地区已设置
- [ ] 适配屏幕尺寸已声明
- [ ] 权限说明已填写
- [ ] API 使用说明已提供

## 隐私政策要求

所有上架的应用都必须提供可访问的隐私政策页面。App 生产工厂内置隐私政策生成功能。

### 通过平台生成

```bash
# 获取自动生成的隐私政策 HTML
# 在项目页面下载或通过 API 获取
GET /api/projects/:id/generated-privacy
```

返回的 HTML 页面包含：

- 信息收集说明
- 信息使用方式
- 信息共享与披露
- 数据存储与安全
- 用户权利
- 隐私政策更新
- 联系方式

### 隐私政策应包含的内容

1. **开发者信息**：公司名称、联系方式
2. **收集的信息类型**：
   - 个人账号信息（如邮箱、手机号）
   - 设备信息（如设备型号、操作系统版本）
   - 使用数据（页面浏览、点击行为）
   - 位置信息（如使用地图功能时）
3. **信息使用目的**：改善服务、个性化推荐、技术运维
4. **第三方共享**：是否与第三方共享数据
5. **数据存储**：存储位置、存储期限、安全措施
6. **用户权利**：查阅、修改、删除个人数据的权利
7. **儿童隐私**：是否收集儿童数据及处理方式
8. **政策更新**：更新方式和通知机制
9. **联系方式**：数据保护负责人联系邮箱

## 发布后维护

### 版本更新

每次修改 App Spec 并重新生成代码后：

1. 更新版本号
2. 在各平台开发者控制台提交新版
3. 更新应用描述（如有变化）

### 用户反馈

- 平台内置项目反馈功能
- API：`POST /api/projects/:id/feedback`

### 数据分析

- **Analytics API** 跟踪用户事件
- **仪表盘** 查看日活、屏幕访问、错误率
- 支持第三方分析工具集成
